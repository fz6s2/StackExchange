Краткое описание.
Оформлением web-страниц не занимался, только бэк.
Может показаться, что создал слишком много классов для решения простой задачи, 
но я старался показать свои навыки и сделать общее расширяемое решение.

Web.
Модель MVC, один сервлет отвечает за все запросы пользователя (ServletController.class). 
Выделены объекты-команды для обработки запросов (Action.class).
Результаты выводятся в jsp (\web\WEB-INF\jsp).
Свои теги jstl в \web\WEB-INF\tlds\tags.tld.

Настройки.
Получение настроек приложения - AppConfig.class.
В настройках содержатся (файл \src\java\ru\inp\stackexchange\resources\config.properties):
- параметры доступа к api Stack Exchange
- параметры прокси
- коды и адреса страниц для навигации внутри сервлета

JSON
Использована билиотека jackson. Парсинг происходит автоматически на основе аннотаций в классах.
Создал пару классов для работы с JSON, в т.ч. преобразование unix time даты в LocalDateTime.

StackExchange
Старался сделать более общее абстрактное решение.
Поэтому выделил несколько классов, не зависимых от содержания Stack Exchange URL.
Запрос к Stack Exchange формируется через построение SERequest и отправку его через SEGatewayClient.
Ответ оборачивается в SEResponse<SEQuestion>,  который содержит список результатов List<Question>.


Логгирование
SLF4J Logback. Файл настроек - \src\java\ru\inp\stackexchange\resources\logback.xml.
